# Security Policy

## Supported Versions

Security updates are provided for:

| Version | Supported |
| --- | --- |
| `main` | ✅ |
| older branches/tags | ❌ |

## Reporting a Vulnerability

Please do **not** open public issues for vulnerabilities.

Instead, report privately with:

- A clear description of the issue
- Reproduction steps / proof of concept
- Impact assessment
- Suggested remediation (if available)

Preferred channels:

1. GitHub private vulnerability reporting (Security Advisories)
2. Maintainer contact: `sharifmdyousuf007@gmail.com`

## Response Process

- Initial acknowledgment target: within 72 hours
- We will triage, validate, and share fix timelines
- After a fix is released, we may publish a disclosure summary

## Scope Notes

Please include any findings related to:

- Admin authentication/authorization
- Convex environment variable handling
- Secret/token leakage
- SSR/client exposure of sensitive data
- Dependency vulnerabilities affecting production
